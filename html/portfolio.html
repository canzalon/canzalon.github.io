string-adt-array (https://github.com/canzalon/string-adt-array)

An implementation of a character string abstract data type, using a dynamic array internally.


string-adt-list (https://github.com/canzalon/string-adt-list)

An implementation of a character string abstract data type, using a singly-linked list with a header, internally.


circular-queue-list (https://github.com/canzalon/circular-queue-list)

Implementation of a circular queue abstract data type that uses the doubly linked list data structure.


maze-solver (https://github.com/canzalon/maze-solver)

Finds solution path for an 8x8 maze using C++ vectors and a stack implementation. See details in files for more information.


pipes (https://github.com/canzalon/pipes)

The pipes program replicates the behavior of the pipes command (|) in a Unix shell. In this particular case, it replicates the "ls -al | sort -n -k 5" command.


call-stats (https://github.com/canzalon/call-stats)

A simple, early project I had done that calculates the cost of a phone call based on the length of the call and number of relay stations used to transmit the signal. It is a procedural program with no OOP aspects.


call-stats-2 (https://github.com/canzalon/call-stats-2)

A sequal to call-stats, this procedural program adds functionality for storing, searching, adding, and removing records. It utilizes the OOP concept of encapsulation.


call-records (https://github.com/canzalon/call-records)

An extension of the call-stats projects with an OOP architecture and added functionality. It stores, processes, searches, adds, removes, and outputs data on a database of call records. Other features include dynamic re-sizing of the database and file stream support.


mad-libs (https://github.com/canzalon/mad-libs)

Simulates the classic mad libs game by accepting a dictionary file and story file and storing them in C++ vectors, then filling the story in with the dictionary key-value pairs and outputting the story.


algorithm-runtime-analysis (https://github.com/canzalon/algorithm-runtime-analysis)

Insertion, heap, and quick sort algorithms are implemented and used to sort random number arrays of varying length. This is done in order to measure their runtime using the appropriate Windows library functions. Averages of trials are found for each and compared with their theoretical runtimes.


fifteen-puzzle-solver (https://github.com/canzalon/fifteen-puzzle-solver)

Produces a randomized board of 15-puzzle, and determines if it is solvable. If so, it finds the solution and produces the moves required to complete it.


hash-based-employee-system (https://github.com/canzalon/hash-based-employee-system)

Hash-based employee system that uses external chaining with buckets as its collision resolution method for handling employee insertion. Additionally, the system includes various features necessary for maintaining a basic employee database.


sql-statements-spj-database (https://github.com/canzalon/sql-statements-spj-database)

Various Sql SELECT statements operating on the classic Supplier Parts Projects (SPJ) Database. See files for details of what each query does, and for the data itself.


sql-system-html-interface (https://github.com/canzalon/sql-system-html-interface)

An interactive SQL system with an html interface that allows for the modification of project records in the SPJ database. Includes the use of procedures, triggers, views, and simple select statements.


embedded-sql-proc-system-html-interface (https://github.com/canzalon/embedded-sql-proc-system-html-interface)

An interactive, browser-based, stand-alone, general purpose, embedded SQL system with an html front-end for the Supplier Parts Projects (SPJ) database. The actions for the HTML forms are Pro*C CGI scripts that carry out the requests with the database.


craps-app (https://github.com/canzalon/craps-app)

A game of craps with it's functionality programmed in javascript, presented with HTML and CSS. All of the aforementioned code is in one html file, "crapsapp.html".


ajax-web-doc-reader (https://github.com/canzalon/ajax-web-doc-reader)

A simple document reader that asynchronously delivers html files, stored in the directory of the website, to the html/javascript front-end upon request. A php file on the server that the website is hosted on fulfills the JavaScript GET requests.


sql-ajax-web-doc-reader (https://github.com/canzalon/sql-ajax-web-doc-reader)

A simple document reader that asynchronously delivers html files, stored in a database, to the html/javascript front-end upon request. The data is dynamically updated via the XMLHttpRequest object communicating with the DOM. The JavaScript GET requests are fulfilled by PHP on the server.

A database project similar to the other .NET projects that instead uses the ADO.NET C# libraries, specifically LINQ, to create a front-end database client for the SPJ database.

The projects in this solution demonstrate some of the basic LINQ with C# capabilities of the .NET framework. Additionally, they exercise the component-based ability of a .NET assembly to be made up of multiple modules.

A database project containing SQL and Transact-SQL scripts of routines (three procedures and an inline function) and the testing of these routines, for the database.


.NET-asp.net-web-applications (https://github.com/canzalon/.NET-asp.net-web-applications)

Solution uses ASP.NET libraries from the .NET framework to create three dynamic web application projects and a web service.
